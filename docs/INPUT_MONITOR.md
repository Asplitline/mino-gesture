# 输入监控功能说明

## 功能概述

实时监控并显示键盘和鼠标输入事件,帮助用户:
- 查看当前按下的所有键和鼠标按钮
- 查看最近的输入事件历史
- 测试输入设备是否正常工作

## 界面说明

### 1. 当前按下区域

显示当前正在按下的键盘按键和鼠标按钮:

- **键盘区**: 显示所有当前按下的键,支持组合键(如 Cmd+C)
- **鼠标区**: 显示当前按下的鼠标按钮(左键/中键/右键)

### 2. 事件日志

记录最近 50 条输入事件:

- **键盘事件**: 显示按键名称,释放时显示 `↑` 符号
- **鼠标事件**: 显示按钮名称(左键/中键/右键)
- **时间戳**: 精确到毫秒的事件时间

## 技术实现

### 后端 (Rust)

**文件**: `src-tauri/src/platform/mouse_listener.rs`

1. **事件捕获**: 使用 `rdev` 库监听全局输入事件
2. **事件类型**:
   - `KeyPress` / `KeyRelease`: 键盘事件
   - `ButtonPress` / `ButtonRelease`: 鼠标事件
3. **事件发送**: 通过 Tauri 的 `emit` API 发送到前端

```rust
#[derive(Debug, Clone, Serialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum InputEvent {
    KeyPress { key: String, code: Option<u32> },
    KeyRelease { key: String, code: Option<u32> },
    MousePress { button: String },
    MouseRelease { button: String },
}
```

### 前端 (React + TypeScript)

**文件**: `src/App.tsx`

1. **事件监听**: 使用 `@tauri-apps/api/event` 监听 `input-event` 事件
2. **状态管理**:
   - `currentKeys`: 当前按下的键集合
   - `currentButtons`: 当前按下的鼠标按钮集合
   - `events`: 事件日志数组(最多50条)
3. **UI 组件**:
   - 实时状态显示区
   - 滚动事件日志
   - 清空按钮

## 使用场景

### 1. 测试输入设备

验证键盘和鼠标是否正常工作,特别是:
- 特殊键(Fn, Cmd, Option, Control)
- 鼠标中键
- 组合键

### 2. 调试手势功能

查看中键手势是否正确触发:
- 按下中键时应该显示"中键"
- 移动鼠标不会产生额外的按键事件
- 释放中键时显示"中键 ↑"

### 3. 学习键盘布局

查看不同键盘布局下的键位名称,特别是:
- 非 QWERTY 布局
- 数字小键盘
- 功能键 (F13-F24)

## 性能优化

1. **事件节流**: 仅记录按键事件,不记录鼠标移动
2. **日志限制**: 只保留最近 50 条记录
3. **高效更新**: 使用 Set 管理当前按下的键,避免重复

## 已知限制

1. **鼠标移动**: 不记录鼠标移动轨迹(性能考虑)
2. **特殊键**: 某些系统级快捷键可能无法捕获(需要辅助功能权限)
3. **输入法**: 中文输入法的候选词不会显示,只显示物理按键

## 权限要求

macOS 需要授予以下权限:

- **辅助功能** (Accessibility): 必需,用于全局事件监听
- **输入监控** (Input Monitoring): 推荐,提高稳定性

权限设置路径:
```
系统设置 > 隐私与安全性 > 辅助功能
系统设置 > 隐私与安全性 > 输入监控
```

## 故障排查

### 按下键盘无反应

1. 检查是否授予辅助功能权限
2. 重启应用
3. 查看终端日志是否有 "rdev listener is receiving events"

### 只显示部分按键

某些按键可能被映射为 `Unknown(code)`,这是正常现象。可以:
1. 查看 code 数值
2. 在 `key_to_string` 函数中添加对应的映射

### 中键不工作

1. 确认鼠标有物理中键(不是触控板)
2. 检查鼠标驱动设置
3. 在其他应用测试中键是否工作
